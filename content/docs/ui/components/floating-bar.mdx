## Usage

The component stores its state in data attributes (`data-scrolled` and `data-hidden`), allowing you to style different states without modifying the component itself.

```tsx
import { FloatingBar } from "@/components/ui/floating-bar";
```

```tsx
<FloatingBar
  position="fixed"
  autoHide
  className="bg-background transition duration-300 data-hidden:-translate-y-full data-scrolled:shadow-lg"
>
  <div className="w-full flex items-center justify-between py-4">
    <div>Logo</div>
    <nav>
      <ul className="flex gap-2">
        <li>
          <a href="#">Home</a>
        </li>
        <li>
          <a href="#">about</a>
        </li>
        <li>
          <a href="#">contact</a>
        </li>
      </ul>
    </div>
  </div>
</FloatingBar>
```

## Examples

### Advanced styling with data attributes

The component automatically manages `data-scrolled` and `data-hidden` attributes based on scroll behavior. You can combine these attributes with Tailwind's data attribute variants to create sophisticated styling:

```tsx
import { FloatingBar } from "@/components/ui/floating-bar";
import { cn } from "@/lib/utils";
```

```tsx
<FloatingBar
  position="fixed"
  autoHide
  className={cn(
    // Base styles
    "flex h-16 w-full items-center justify-between px-6",
    // Transition
    "transform-gpu transition duration-300",
    // Initial state
    "bg-transparent backdrop-blur-2xl",
    // Scrolled state - when the user starts scrolling
    "data-scrolled:bg-background data-scrolled:shadow-lg",
    // Hidden state for auto-hide behavior
    "data-hidden:data-scrolled:shadow-none data-hidden:motion-safe:-translate-y-full"
  )}
>
  <div className="font-semibold">My App</div>
  <nav>
    <ul className="flex gap-6">
      <li>
        <a href="#" className="hover:text-primary">
          Home
        </a>
      </li>
      <li>
        <a href="#" className="hover:text-primary">
          About
        </a>
      </li>
      <li>
        <a href="#" className="hover:text-primary">
          Contact
        </a>
      </li>
    </ul>
  </nav>
</FloatingBar>
```

The styling works by targeting data attributes:
- `data-scrolled:bg-background` - Applied when scroll position exceeds `scrolledThreshold`
- `data-scrolled:shadow-lg` - Adds shadow when scrolled
- `data-hidden:motion-safe:-translate-y-full` - Slides the bar up when auto-hidden (respecting motion preferences)
- `data-hidden:data-scrolled:shadow-none` - Combines both states to remove shadow when hidden
- `transform-gpu transition duration-300` - Provides smooth GPU-accelerated animations

All state management happens internally via data attributes, keeping your styling declarative and the component unmodified.
